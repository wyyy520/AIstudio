# AIStudio AI 对话页面设计规范

# 页面定位

**AI 对话页面是 AIStudio 内置的智能助手对话界面，参考 Cursor AI 交互风格。**

主要功能：
- 用户可以用自然语言描述需求，AI 自动生成工作流
- AI 帮你分析错误，自动修复环境问题
- AI 帮你解释节点功能，推荐节点组合
- AI 可以分析日志，找出问题原因

这是 AIStudio 的智能交互入口，必须做到 **Cursor** 级别的流畅体验。

---

# 用户目标

| 用户 | 目标 |
|------|------|
| AI 初学者 | 自然语言描述需求，AI 自动生成可运行的工作流 |
| 开发者遇到问题 | AI 帮助分析错误，给出解决方案 |
| 学习使用 | AI 解释概念，推荐最佳实践 |
| 快速原型 | 快速生成工作流原型，再手动调整 |

# 页面布局

参考 **Cursor** 对话框设计：

```
┌─────────────────────────────────────────────────────────────┐
│  AI 助手                                    [清空] [设置]     │  ← 顶部标题栏
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                         对话历史                             │
│                                                             │
│  用户消息  →  靠右气泡                                        │
│                                                             │
│          AI 回复  ←  靠左气泡                                │
│                                                             │
│                         ...                                 │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  [附件节点]  输入框...                      [发送按钮] [快捷键]│  ← 底部输入区
└─────────────────────────────────────────────────────────────┘
```

## 布局特点

- 全屏对话模式
- 消息列表占满大部分空间
- 输入框固定在底部
- 新消息自动滚动到底部
- 支持拖拽调整输入框高度

# 页面结构

## 1. 顶部标题栏

**内容：**
- 左侧："AI 助手" 标题 + 模型名称标签
- 右侧：
  - 清空对话按钮
  - AI 设置按钮（选择模型、配置参数）
  - 关闭按钮

**样式：**
- 高度：48px
- 背景：`bg-secondary`
- 边框底部：`border-subtle`

---

## 2. 消息列表区域

**内容：**
- 按时间顺序倒序？不，正序：最早在上，最新在下
- 每条消息：
  - 用户消息：靠右，主色背景
  - AI 消息：靠左，二级背景

**消息气泡内容：**
- 用户消息：纯文本
- AI 消息：支持 Markdown 渲染
  - 代码块高亮
  - 列表
  - 链接
- AI 生成工作流卡片：
  - 工作流预览图
  - 名称描述
  - "应用到画布" / "另存为新工作流" 按钮

**滚动：**
- 新消息自动平滑滚动到底部
- 用户滚动查看历史时，不自动滚动

---

## 3. 上下文提示条

**当在工作流编辑器打开 AI 对话时：**

顶部显示上下文提示条：
- "当前上下文：工作流 [xxx]"
- "已选中节点：[xxx]"
- 帮助 AI 理解用户当前在做什么

---

## 4. 底部输入区域

**结构：**
```
┌─────────────────────────────────────────────────────────┐
│ [附件]  [节点]                                                 │  ← 附件按钮行
├─────────────────────────────────────────────────────────┤
│                                                 [发送]   │  ← 输入框+按钮
└─────────────────────────────────────────────────────────┘
```

**内容：**
- 附件按钮：可以附加当前工作流 / 选中节点 / 日志给 AI
- 输入框：支持多行输入，高度自适应（最小 40px，最大 200px）
- 发送按钮：点击发送，Enter 发送，Ctrl+Enter 换行
- 右下角显示"快捷键 Ctrl+Shift+L 打开对话框"提示（小字）

**已附件提示：**
- 标签显示已附加的内容
- 点击 X 可移除附件

---

## 5. AI 生成工作流卡片

当 AI 生成了工作流，在消息中显示：

```
┌─────────────────────────────────────────────────────┐
│  ✓ AI 已为你生成工作流：图像分类演示                  │
│                                                     │
│  [缩略图预览]                                      │
│  节点数：8 个                                      │
│                                                     │
│  [在当前画布打开]  [另存为新工作流]  [查看代码]     │
└─────────────────────────────────────────────────────┘
```

**样式：**
- 圆角 10px
- 边框：主色 1px
- 背景：`primary-bg`

**交互：**
- 点击"在当前画布打开" → 替换当前画布或追加
- 点击"另存为新工作流" → 保存并跳转到编辑器

# 页面组件

## 1. 消息气泡组件

### 用户消息

- 靠右对齐
- 圆角 16px（左上 16px，右上 4px，左下 16px，右下 16px）
- 背景：`primary`
- 文字：白色
- 最大宽度：70%

### AI 消息

- 靠左对齐
- 圆角 16px（左上 4px，右上 16px，左下 16px，右下 16px）
- 背景：`bg-tertiary`
- 文字：`text-primary`
- 最大宽度：85%

### 流式输出

- AI 回复时逐字输出
- 光标闪烁动画
- 用户可以在输出过程中打断

---

## 2. 代码块组件

- 圆角 8px
- 背景：比气泡背景深一点
- 语法高亮（对应语言）
- 右上角：复制代码按钮
- 使用等宽字体

---

## 3. 附件标签组件

- 圆角 6px
- 背景：`bg-active`
- 内容：图标 + 名称 + × 按钮
- 高度：24px
- 点击 × 移除附件

---

# UI规范

- 整体风格参考 **Cursor** AI 对话框
- 留白充足，消息之间间距 16px
- 气泡圆角设计符合整体规范
- Markdown 渲染清晰，层级分明
- 代码块有足够的对比度，方便阅读
- 流式输出过程中，已经输出的文字稳定，不跳动

---

# 交互规范

| 操作 | 行为 |
|------|------|
| Enter | 发送消息 |
| Ctrl+Enter | 换行 |
| 点击附件按钮 | 弹出附件选项菜单 |
| 滚动消息列表 | 停止自动滚动到底部（直到用户手动滚到底） |
| 点击代码块复制按钮 | 复制到剪贴板，显示"已复制"提示 |
| 流式输出时点击停止 | 立即停止生成 |

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Shift+L` | 打开/聚焦 AI 对话框 |
| `Esc` | 关闭对话框 |

---

# 动画规范

| 动画 | 时长 | 说明 |
|------|------|------|
| 新消息滑入 | 200ms | 从下方滑入淡入 |
| 流式逐字输出 | 即时 | 每个字符逐个出现 |
| 输入框高度变化 | 200ms | 自适应高度平滑变化 |
| 滚动到底部 | 平滑 | 平滑滚动，不是跳变 |
| 发送按钮悬停 | 150ms | 背景颜色变化 |

---

# 状态设计

## 输入状态

| 状态 | 表现 |
|------|------|
| 空输入 | 发送按钮禁用 |
| 输入中 | 发送按钮启用 |
| AI 正在生成 | 输入框禁用，发送按钮变为停止按钮 |

## 消息状态

| 状态 | 表现 |
|------|------|
| 发送中 | 气泡显示骨架屏，闪烁加载 |
| 流式输出 | 逐字显示，光标闪烁 |
| 完成 | 完整显示，可交互 |
| 发送失败 | 红色提示，重试按钮 |

---

# 错误状态

## API 调用失败

- AI 消息显示错误信息
- 红色提示
- 重试按钮
- 常见错误原因：
  - API Key 未配置
  - 配额不足
  - 网络错误

## 生成工作流失败

- 说明错误原因
- 询问用户是否需要重新生成
- 提供修改建议

# 加载状态

- **发送中**：用户消息显示后，AI 消息位置显示骨架屏 + 加载动画
- **流式输出中**：已输出文字正常显示，末尾有光标闪烁
- **页面加载**：消息列表显示过渡动画

---

# 页面跳转关系

- **从工作流编辑器 →** 点击 AI 助手按钮打开此页面（侧边面板或弹窗）
- **从仪表盘 →** 点击 AI 助手入口打开
- **从这里生成工作流 →** 跳转到工作流编辑器
- **从这里 →** 关闭返回上一个页面

---

# API交互

| API | 用途 |
|-----|------|
| `chat/send` | 发送消息，返回流式/非流式响应 |
| `chat/stop` | 停止当前生成 |
| `chat/history` | 获取对话历史 |
| `chat/clear` | 清空对话历史 |
| `chat/apply-workflow` | 应用 AI 生成的工作流到画布 |
| `chat/attachments` | 获取可附加的上下文选项 |

# 后续扩展

1. **多模型支持**：支持切换不同 AI 模型（GPT-4、Claude、本地模型等）
2. **对话历史**：支持保存和加载历史对话
3. **多轮对话优化**：自动保留上下文，理解之前对话内容
4. **知识库集成**：支持上传本地文档作为 AI 的参考知识
5. **语音输入**：支持语音转文字输入
6. **图片输入**：支持上传图片让 AI 分析
