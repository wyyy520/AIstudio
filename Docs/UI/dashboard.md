# AIStudio 仪表盘页面设计规范

# 页面定位

**仪表盘是用户打开 AIStudio 后看到的第一个页面。**

它提供：
- 项目快速访问
- 最近编辑的工作流
- 快速开始入口
- 系统状态概览
- 公告和更新信息

整体风格参考 VS Code 欢迎页面，简洁专业。

---

# 用户目标

| 用户 | 目标 |
|------|------|
| 新用户 | 快速找到创建新项目的入口，开始使用 |
| 老用户 | 快速打开最近编辑的项目/工作流，继续工作 |
| 所有用户 | 了解系统状态（环境、GPU、存储） |

# 页面结构

## 1. 顶部导航栏

**内容：**
- 左侧：AIStudio Logo + 名称
- 右侧：
  - 快速搜索（搜索项目、工作流、节点）
  - 主题切换按钮
  - 设置按钮
  - 用户头像/菜单

**样式：**
- 高度：64px
- 背景：透明（与主背景一致）
- 没有边框

---

## 2. 快速开始卡片

**位置**：左上

**内容：**
- 标题：快速开始
- 四个操作按钮：
  1. 新建项目 → 打开新建项目对话框
  2. 打开项目 → 打开文件选择对话框
  3. 导入项目 → 导入压缩包/JSON
  4. 浏览模板 → 打开模板中心

**按钮样式：**
- 图标 + 文字垂直排列
- 圆角 10px
- 悬停背景变化

---

## 3. 最近编辑卡片

**位置**：右上

**内容：**
- 标题：最近编辑
- 列表项：
  - 工作流/项目名称
  - 修改时间（相对时间：今天、昨天、3天前）
  - 所属项目
- 底部："查看全部" 链接 → 跳转到项目管理页面

**列表项交互：**
- 点击 → 打开工作流编辑器
- 右键 → 菜单（打开项目位置、复制路径、移除历史）

---

## 4. 系统状态卡片

**位置**：左下

**内容：**
- 标题：系统状态
- 状态项：
  - Python 环境：正常 / 异常 + 版本号
  - GPU 状态：空闲 / 繁忙 + 显存使用
  - CUDA 版本：已安装 / 未安装
  - 磁盘使用：已用 / 总量
  - 已安装插件数量
- 底部按钮："检查环境" → 运行环境自动检查和修复

**状态指示：**
- 正常：绿色小圆 + "正常"
- 警告：黄色小圆 + "警告"
- 错误：红色小圆 + "错误"

---

## 5. 公告/更新卡片

**位置**：右下

**内容：**
- 标题：最新公告
- 当前版本号
- 更新内容列表（精简，3 条以内）
- 底部："查看全部更新日志" 链接
- 如果没有新公告，显示欢迎语和帮助文档链接

---

# 页面组件

## 1. 状态指示器组件

- 圆形指示器（直径 10px）
- 颜色对应状态（绿/黄/红）
- 右侧显示状态文字

## 2. 最近项目项组件

- 高度：48px
- 圆角 8px
- 悬停：`bg-hover`
- 左侧：文件类型图标
- 中间：名称 + 次要文字（修改时间）
- 右侧：更多操作按钮

## 3. 快捷操作按钮组件

- 宽高：80px × 80px
- 圆角 10px
- 背景：`bg-tertiary`
- 图标在上（32px）
- 文字在下（13px）
- 悬停：背景变 `bg-hover`，上移 2px

---

# UI规范

- 所有卡片圆角 12px
- 所有卡片背景 `bg-tertiary`
- 卡片间距 16px
- 内容内边距 20px
- 整体留白充分，不拥挤
- 重要操作按钮突出显示

---

# 交互规范

- 卡片悬停：阴影轻微增强
- 列表项点击：立即跳转
- 最近项右键菜单：弹出操作菜单
- 搜索框聚焦：展开宽度，显示搜索结果下拉

---

# 动画规范

- 页面加载：卡片依次淡入，从上到下
- 卡片悬停：阴影变化 150ms，快捷按钮轻微上移
- 导航栏操作：下拉菜单淡入

---

# 状态设计

## 正常状态

- 所有数据加载完成，正常显示

## 加载中状态

- 骨架屏占位
- 脉动动画
- 不可交互

---

# 错误状态

**当系统状态检查失败：**

- 对应状态项变红
- 显示"修复"按钮
- 点击按钮调用 AI 自动修复环境功能
- 修复过程显示进度

---

# 空页面设计

**当没有最近编辑记录：**

- 最近卡片显示提示文字："还没有编辑记录，开始创建第一个项目吧"
- 文字颜色 `text-tertiary`
- 居中显示

**当没有公告：**

- 显示欢迎信息 + 快速链接（文档、教程、社区）

---

# 加载状态

- 页面骨架屏预加载
- 数据加载完成后替换骨架，淡入效果
- 系统状态异步加载，先占位后更新

---

# 页面跳转关系

- **应用启动 →** 进入此页面
- **新建项目按钮 →** 打开新建项目弹窗 → 创建后跳转到项目管理
- **打开项目 →** 文件选择 → 打开后跳转到项目管理
- **最近编辑项点击 →** 直接跳转到工作流编辑器
- **查看全部 →** 跳转到项目管理页面
- **检查环境 →** 打开环境检查对话框
- **设置按钮 →** 跳转到设置页面
- **插件市场 →** 跳转到插件市场页面

---

# API交互

| API | 用途 |
|-----|------|
| `dashboard/recent` | 获取最近编辑列表 |
| `dashboard/system-status` | 获取系统状态 |
| `dashboard/announcements` | 获取公告列表 |
| `environment/check` | 触发环境检查 |
| `environment/repair` | 触发自动修复 |
