# AIStudio 工作流编辑器页面设计规范

# 页面定位

**工作流编辑器是 AIStudio 的核心页面，是用户编排 AI 工作流的主要场所。**

用户在这里通过拖拽节点、连接端口的方式可视化编排 AI 处理流程，支持实时预览、调试和运行。

这是整个软件最重要的核心功能，必须达到 **Simulink + ComfyUI** 的专业水准。

---

# 用户目标

| 用户角色 | 用户目标 |
|----------|----------|
| AI 初学者 | 通过拖拽方式快速搭建第一个 AI 工作流，理解 AI 处理流程 |
| 算法工程师 | 快速迭代实验不同的模型组合，验证算法思路 |
| 科研人员 | 编排复杂的数据分析流程，可复现实验结果 |
| 企业开发 | 构建生产可用的 AI 推理 pipeline，部署到生产环境 |

# 页面结构

## 区域说明

### 1. 顶部工具栏

**位置**：页面最顶部

**内容**：
- 左侧：面包屑导航（项目名称 / 工作流名称）
- 中间：操作按钮（保存、运行、暂停、停止、调试）
- 右侧：视图控制（缩小、放大、自适应、全屏）

**样式**：
- 背景：`bg-secondary`
- 高度：48px
- 圆角：无（顶部全屏宽）

---

### 2. 左侧节点面板

**位置**：左侧边栏

**内容**：
- 搜索框（搜索节点）
- 分类折叠面板：
  - 视觉处理（Vision）
  - 自然语言处理（NLP）
  - 时序处理（TimeSeries）
  - 逻辑控制（Logic）
  - 系统工具（System）
  - MCP 服务
  - 仿真联动（Simulation）
  - Agent

每个节点项：
- 节点图标（对应类型颜色）
- 节点名称
- 简短描述（hover 显示完整）

**交互**：
- 拖拽节点到画布创建节点
- 点击节点后点击画布创建节点
- 搜索实时过滤节点列表
- 分类可折叠

---

### 3. 中间画布区域

**位置**：中心区域

**内容**：
- 无限画布背景
- 网格线（浅色，低透明度）
- 节点集合
- 连线集合
- 迷你地图（右上角）

**背景样式**：
- 深色主题：`bg-primary`
- 网格线：`rgba(255, 255, 255, 0.05)`
- 网格大小：20px × 20px

**功能**：
- 拖拽画布平移
- 滚轮缩放（25% ~ 200%）
- 框选多选节点
- 空白处右键菜单
- 支持快捷键操作

---

### 4. 右侧属性面板

**位置**：右侧边栏（参考 Figma）

**内容**（当节点未选中时）：
- 工作流名称输入框
- 工作流描述
- 运行配置
- 全局变量设置
- 工作流统计（节点数、连接数）

**内容**（当节点选中时）：
- 节点名称编辑
- 节点类型显示（带颜色标签）
- 节点描述
- 动态参数表单（根据节点类型生成）
- 输入端口配置
- 输出端口配置
- 高级设置
- 删除按钮

**样式**：
- 参数分组使用分割线分隔
- 每个参数有标签 + 输入控件
- 滚动内容超出时滚动

---

### 5. 底部控制台

**位置**：底部

**内容**：
- 标签页（日志 / 节点输出 / 问题）
- 日志输出区域（时间戳 + 级别 + 消息）
- 清除按钮
- 自动滚动开关

**日志级别颜色**：
- Info：`text-secondary`
- Success：`success`
- Warning：`warning`
- Error：`error`

# 页面组件

## 1. 工作流节点组件

### 外观（参考 Simulink + ComfyUI）

```
┌─────────────────────────────────────────────┐
│  [图标] 节点标题                ● 状态指示器 │  ← 头部
├─────────────────────────────────────────────┤
│                                             │
│  输入端口  ▶  参数预览                         │  ← 内容区
│                                             │
│                                             │
├─────────────────────────────────────────────┤
│                                             │  ← 输出端口
│  输出端口                                    │
└─────────────────────────────────────────────┘
```

**尺寸：**
- 最小宽度：180px
- 自适应内容高度
- 圆角：10px
- 阴影：`shadow-node`

**颜色：**
- 边框左侧有 4px 宽的类型颜色边条
- 背景：`bg-tertiary`
- 头部背景：比主体深一点点

**状态指示器：**
- Idle：灰色小圆
- Running：黄色小圆 + 脉冲动画
- Success：绿色小圆
- Error：红色小圆

---

### 节点端口

**位置：**
- 输入端口：在节点左侧
- 输出端口：在节点右侧

**样式：**
- 圆形手柄，直径 14px
- 边框颜色：对应节点类型颜色
- 填充：背景色
- Hover 时放大到 18px，填充类型颜色

**交互：**
- 拖拽输出端口到输入端口创建连线
- 点击端口开始连线
- Hover 时提示端口名称和数据类型

---

### 节点状态

| 状态 | 视觉效果 |
|------|----------|
| Idle | 正常样式，灰色指示器 |
| Waiting | 浅黄边框，等待执行 |
| Running | 黄色边框，运行动画（脉动），黄色指示器 |
| Success | 绿色边框，绿色指示器 |
| Error | 红色边框，红色指示器，抖动一下提示 |

---

### 节点动画

- **选中**：轻微上浮，阴影增强，边框变主色
- **拖拽中**：半透明（80%），保持跟随鼠标
- **运行中**：背景微妙脉动动画
- **错误**：微小抖动动画，吸引注意力

---

## 2. 连线组件

### 外观

- 贝塞尔曲线连接两个端口
- 线条宽度：2px
- 默认颜色：`border-strong`
- 圆角线条

### 状态

| 状态 | 样式 |
|------|------|
| Idle | `border-strong`，宽度 2px |
| Hover | 变宽到 3px，变主色 |
| Running | 数据流动动画，颜色 `primary` |
| Selected | 变宽到 3px，`primary` 颜色 |
| Error | `error` 颜色 |

### 数据流动动画

当工作流运行时，数据流过连线应有动画效果：
- 沿着连线方向，从输出端口到输入端口
- 渐变光点沿线条移动
- 动画周期 1.5 秒，循环
- 颜色为主色，透明度从 0 到 0.8 到 0

---

## 3. 迷你地图组件

**位置**：画布右下角

**功能**：
- 显示整个工作流的缩略图
- 当前视口矩形框可拖拽导航
- 点击跳转到对应位置
- 可折叠收起

**样式**：
- 半透明背景
- 圆角 8px
- 尺寸：默认 180px × 120px

---

## 4. 上下文菜单

**触发**：右键点击

**位置**：鼠标位置

**内容根据上下文：**

- **空白处右键**：
  - 粘贴
  - 全选
  - 对齐选中节点
  - 框选反向
  - 自适应画布

- **节点右键**：
  - 剪切
  - 复制
  - 粘贴
  - 删除
  - 复制节点 ID
  - 禁用节点
  - 打开文档

**样式：**
- 圆角 8px
- 阴影 `shadow-lg`
- 背景 `bg-secondary`
- 菜单项悬停 `bg-hover`
- 动画：淡入 + 向下滑动 10px

---

## 5. 框选组件

**交互**：鼠标拖拽拉出矩形选框

**样式：**
- 边框：主色 1px
- 背景：主色 10% 透明度
- 圆角：4px

**行为：**
- 框选包含的节点全部选中
- 按住 Shift 添加到选区
- 按住 Alt 从选区减去

# UI规范

## 画布

- 无限画布，支持平移
- 浅色网格线，不干扰视觉
- 深色主题网格透明度 5%，浅色主题 8%
- 迷你地图不占主要空间，放在右下角

## 节点

- 左侧色条标识节点类型，快速识别
- 圆角 10px
- 必须有阴影，和画布区分开
- 端口明显，容易点击
- 状态指示器在右上角，一目了然

## 连线

- 使用贝塞尔曲线，平滑自然
- 不要出现折线拐角
- 数据流动动画必须丝滑
- 运行时高亮，让用户知道当前执行到哪里

## 属性面板

- 参考 Figma 属性面板设计
- 参数分组清晰
- 标签和控件对齐
- 滚动流畅
- 节点未选中时显示工作流级配置

## 节点面板

- 搜索框在顶部，快速查找
- 分类折叠，减少认知负担
- 节点项简洁，只显示图标和名称
- 拖拽反馈清晰

# 交互规范

## 基础操作

| 操作 | 手势 | 行为 |
|------|------|------|
| 平移画布 | 鼠标拖拽空白处 / 中键拖拽 | 画布跟随移动，流畅 |
| 缩放 | 滚轮 / Ctrl + 滚轮 | 以鼠标位置为中心平滑缩放 |
| 创建节点 | 拖拽节点从面板到画布 | 松开鼠标创建节点在鼠标位置 |
| 移动节点 | 拖拽节点 | 节点跟随鼠标，多节点一起移动 |
| 选中节点 | 点击节点 | 节点选中，属性面板显示节点信息 |
| 多选 | 框选 / Shift + 点击 | 多个节点选中 |
| 取消选中 | 点击空白处 | 清除选区，显示工作流配置 |
| 创建连线 | 拖拽输出端口到输入端口 | 松开创建连线 |
| 删除连线 | 选中连线按 Delete | 删除连线 |
| 删除节点 | 选中节点按 Delete | 删除节点和相关连线 |
| 复制节点 | Ctrl+C / Ctrl+V | 在新位置复制节点 |
| 剪切节点 | Ctrl+X | 剪切到剪贴板 |
| 粘贴节点 | Ctrl+V | 粘贴在当前视图中心 |
| 全选 | Ctrl+A | 选中所有节点 |

## 对齐辅助

- 移动节点时，当节点和其他节点边缘对齐时，显示对齐辅助线
- 辅助线颜色：主色，透明度 50%
- 帮助用户快速整齐排列节点

## 连线校验

- 输出不能连输出，输入不能连输入
- 数据类型不匹配时，连线变红提示错误
- 循环连接时提示错误，不允许创建

## 撤销/重做

- 支持 Ctrl+Z 撤销，Ctrl+Shift+Z 重做
- 每一步操作都可撤销，最多保存 50 步

# 状态设计

## 画布状态

| 状态 | 表现 |
|------|------|
| 空闲 | 正常显示，等待操作 |
| 拖拽节点中 | 节点预览跟随鼠标，半透明 |

详见「页面组件 → 工作流节点组件 → 节点状态」

# 错误状态

## 节点错误

- 节点边框变红
- 状态指示器变红
- 错误信息显示在节点底部
- 控制台输出错误详情
- 停止下游节点执行
- 已执行节点保持结果

## 连线错误

- 连线变红
- 鼠标悬停显示错误原因
- 常见错误：类型不匹配、循环依赖

## 工作流错误

- 弹窗提示错误概要
- 错误节点自动定位到画布中心
- 滚动控制台到错误信息

# 页面跳转关系

- **从项目管理页面 →** 点击工作流条目进入此页面
- **从 AI 对话 →** Agent 自动生成工作流后跳转到这里
- **从这里 →** 保存后返回项目管理页面
- **从这里 →** 打开插件市场安装新节点
- **从这里 →** 运行后查看日志在控制台

# API交互

## 主要 API 调用

| 操作 | API 方法 | 说明 |
|------|----------|------|
| 加载工作流 | `workflow/get` | 获取工作流 JSON 数据 |
| 保存工作流 | `workflow/save` | 保存当前工作流 JSON |
| 运行工作流 | `workflow/run` | 开始执行，返回任务 ID |
| 停止运行 | `workflow/stop` | 停止当前执行 |
| 节点列表 | `workflow/nodes` | 获取所有可用节点类型 |
| 验证工作流 | `workflow/validate` | 验证工作流是否合法 |
| 导出工作流 | `workflow/export` | 导出为 JSON 文件 |
| 导入工作流 | `workflow/import` | 从 JSON 导入 |

## 实时状态推送

使用 WebSocket 实时接收：
- `node_status`：节点状态变化
- `node_output`：节点输出数据
- `workflow_progress`：整体进度
- `workflow_done`：完成通知
- `log_message`：日志消息

# 后续扩展
