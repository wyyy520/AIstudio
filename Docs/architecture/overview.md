# AI Studio 总体架构概览

## 1. 系统定位

AI Studio 是一个智能工作流开发平台，通过可视化拖拽编排 AI 能力节点，实现从数据处理、模型推理到自动化部署的全流程开发。

## 3. 核心模块职责

| 模块 | 技术栈 | 核心职责 |
|------|--------|----------|
| Frontend | Vue3 + TypeScript + Tauri | 用户界面、工作流可视化编辑、AI 对话交互 |
| Backend | Go | API 网关、工作流编排、任务调度、Agent 管理、插件加载 |
| Engine | Python | AI 模型推理执行（视觉/NLP/语音/时序） |
| Plugins | 多语言 | 可扩展功能节点（视觉/NLP/逻辑/系统/仿真/MCP） |
| Runtime | - | 运行时缓存、日志、临时文件、任务工作空间 |
| Storage | - | 用户数据持久化（数据集/模型/项目/配置） |

## 4. 通信协议

| 通信路径 | 协议 | 说明 |
|----------|------|------|
| Frontend ↔ Backend | Tauri IPC / WebSocket | 桌面端用 Tauri IPC，Web 端用 WebSocket |
| Backend ↔ Engine | gRPC / HTTP | Go 调用 Python 引擎执行推理 |
| Backend ↔ Plugins | Go Plugin / RPC | 动态加载或远程调用 |
| Backend ↔ MCP | JSON-RPC | MCP 协议通信 |

## 5. 数据流

```
用户操作 → Frontend → Backend(API) → 工作流引擎(解析/调度)
    → 任务队列 → Engine(Python推理) / Plugin(执行)
    → 结果回传 → Backend → Frontend(渲染)
```

## 6. 部署模式

- **桌面模式**：Tauri 打包，Frontend + Backend 内嵌，Engine 本地调用
- **服务模式**：Backend 独立部署，Frontend 通过 HTTP/WebSocket 访问，Engine 远程调用
